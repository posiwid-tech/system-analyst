# Часть 4. Kafka-события

## 4.1. Событие создания бронирования — BookingCreated

Событие `BookingCreated` публикуется сервисом бронирования после успешного создания и сохранения бронирования.

### Структура сообщения

| Поле          | Тип данных                   | Обязательный | Описание                                                             |
| ------------- | ---------------------------- | ------------ | -------------------------------------------------------------------- |
| event_id      | UUID                         | да           | Уникальный идентификатор события                                     |
| event_type    | string                       | да           | Тип события. Для данного события принимает значение `BookingCreated` |
| event_version | integer                      | да           | Версия схемы события                                                 |
| event_time    | string (date-time, ISO 8601) | да           | Дата и время возникновения события                                   |
| payload       | object                       | да           | Данные предметной области, связанные с созданным бронированием       |

### Payload

| Поле         | Тип данных                   | Обязательный | Описание                                                        |
| ------------ | ---------------------------- | ------------ | --------------------------------------------------------------- |
| booking_id   | UUID                         | да           | Уникальный идентификатор бронирования                           |
| user_id      | UUID                         | да           | Уникальный идентификатор пользователя, создавшего бронирование  |
| workspace_id | UUID                         | да           | Уникальный идентификатор забронированного рабочего пространства |
| start_time   | string (date-time, ISO 8601) | да           | Дата и время начала бронирования                                |
| end_time     | string (date-time, ISO 8601) | да           | Дата и время окончания бронирования                             |
| price        | number                       | да           | Стоимость бронирования, зафиксированная на момент его создания  |
| currency     | string                       | да           | Валюта стоимости бронирования                                   |
| status       | string                       | да           | Статус бронирования после создания                              |

### Пример сообщения

```json
{
  "event_id": "de487d89-f968-44b3-989c-a3398821dc52",
  "event_type": "BookingCreated",
  "event_version": 1,
  "event_time": "2026-07-20T14:32:15+03:00",
  "payload": {
    "booking_id": "b52cb10b-ef43-4f23-b226-cf44c93f73c1",
    "user_id": "415f5742-343b-4a54-86e9-f7d9fab269fd",
    "workspace_id": "8a12f316-3b1b-4b5f-9b8c-f4d638a7f125",
    "start_time": "2026-07-25T10:00:00+03:00",
    "end_time": "2026-07-25T12:00:00+03:00",
    "price": 2500.00,
    "currency": "RUB",
    "status": "CONFIRMED"
  }
}
```

## 4.2. Событие отмены бронирования — BookingCancelled

Событие `BookingCancelled` публикуется сервисом бронирования после успешного изменения статуса бронирования на `CANCELLED`.

### Структура сообщения

| Поле          | Тип данных                   | Обязательный | Описание                                                               |
| ------------- | ---------------------------- | ------------ | ---------------------------------------------------------------------- |
| event_id      | UUID                         | да           | Уникальный идентификатор события                                       |
| event_type    | string                       | да           | Тип события. Для данного события принимает значение `BookingCancelled` |
| event_version | integer                      | да           | Версия схемы события                                                   |
| event_time    | string (date-time, ISO 8601) | да           | Дата и время возникновения события                                     |
| payload       | object                       | да           | Данные предметной области, связанные с отменённым бронированием        |

### Payload

| Поле         | Тип данных                   | Обязательный | Описание                                                                 |
| ------------ | ---------------------------- | ------------ | ------------------------------------------------------------------------ |
| booking_id   | UUID                         | да           | Уникальный идентификатор отменённого бронирования                        |
| user_id      | UUID                         | да           | Уникальный идентификатор пользователя, которому принадлежит бронирование |
| workspace_id | UUID                         | да           | Уникальный идентификатор рабочего пространства                           |
| start_time   | string (date-time, ISO 8601) | да           | Дата и время начала отменённого бронирования                             |
| end_time     | string (date-time, ISO 8601) | да           | Дата и время окончания отменённого бронирования                          |
| status       | string                       | да           | Новый статус бронирования. Принимает значение `CANCELLED`                |
| cancelled_at | string (date-time, ISO 8601) | да           | Дата и время отмены бронирования                                         |

### Пример сообщения

```json
{
  "event_id": "780b6722-22c4-4149-a725-68193bb75551",
  "event_type": "BookingCancelled",
  "event_version": 1,
  "event_time": "2026-07-22T09:15:37+03:00",
  "payload": {
    "booking_id": "b52cb10b-ef43-4f23-b226-cf44c93f73c1",
    "user_id": "415f5742-343b-4a54-86e9-f7d9fab269fd",
    "workspace_id": "8a12f316-3b1b-4b5f-9b8c-f4d638a7f125",
    "start_time": "2026-07-25T10:00:00+03:00",
    "end_time": "2026-07-25T12:00:00+03:00",
    "status": "CANCELLED",
    "cancelled_at": "2026-07-22T09:15:37+03:00"
  }
}
```

## Публикация событий

Продюсером событий `BookingCreated` и `BookingCancelled` является `Booking Service`.

Событие `BookingCreated` публикуется только после успешного сохранения нового бронирования.

Событие `BookingCancelled` публикуется только после успешного изменения состояния существующего бронирования на `CANCELLED`.

Потребителями событий могут быть:

* система уведомлений;
* сервис аналитики;
* другие компоненты системы, которым необходимо реагировать на изменение жизненного цикла бронирования.

В качестве ключа Kafka-сообщения рекомендуется использовать `booking_id`. Это позволяет сообщениям, относящимся к одному бронированию, попадать в одну партицию Kafka и сохранять порядок их обработки в рамках данного бронирования.
