# Часть 3. REST API

## 3.1. Создание бронирования

### Общая информация

| Поле              | Описание                                                                              |
| ----------------- | ------------------------------------------------------------------------------------- |
| Endpoint name     | CreateBooking                                                                         |
| HTTP method       | POST                                                                                  |
| URL               | `/api/v1/bookings`                                                                    |
| Описание сценария | Создаёт бронирование выбранного рабочего пространства на указанный временной интервал |

### Входящие данные

| Параметр        | Тип    | Тип данных                   | Обязательный | Описание                                                                    |
| --------------- | ------ | ---------------------------- | ------------ | --------------------------------------------------------------------------- |
| Authorization   | Header | string                       | да           | Токен авторизации пользователя                                              |
| Idempotency-Key | Header | string                       | да           | Уникальный ключ запроса для предотвращения повторного создания бронирования |
| workspace_id    | Body   | UUID                         | да           | Уникальный идентификатор рабочего пространства                              |
| start_time      | Body   | string (date-time, ISO 8601) | да           | Дата и время начала бронирования                                            |
| end_time        | Body   | string (date-time, ISO 8601) | да           | Дата и время окончания бронирования                                         |

Идентификатор пользователя определяется системой на основании данных аутентификации и не передаётся клиентом в теле запроса.

### Пример запроса

```json
{
  "workspace_id": "8a12f316-3b1b-4b5f-9b8c-f4d638a7f125",
  "start_time": "2026-07-25T10:00:00+03:00",
  "end_time": "2026-07-25T12:00:00+03:00"
}
```

### Успешный ответ

HTTP Status: `201 Created`

| Параметр     | Тип данных         | Обязательный | Описание                                             |
| ------------ | ------------------ | ------------ | ---------------------------------------------------- |
| booking_id   | UUID               | да           | Уникальный идентификатор созданного бронирования     |
| workspace_id | UUID               | да           | Идентификатор забронированного рабочего пространства |
| start_time   | string (date-time) | да           | Дата и время начала бронирования                     |
| end_time     | string (date-time) | да           | Дата и время окончания бронирования                  |
| price        | number             | да           | Зафиксированная стоимость бронирования               |
| currency     | string             | да           | Валюта стоимости                                     |
| status       | string             | да           | Текущий статус бронирования                          |
| created_at   | string (date-time) | да           | Дата и время создания бронирования                   |

### Пример успешного ответа

```json
{
  "booking_id": "b52cb10b-ef43-4f23-b226-cf44c93f73c1",
  "workspace_id": "8a12f316-3b1b-4b5f-9b8c-f4d638a7f125",
  "start_time": "2026-07-25T10:00:00+03:00",
  "end_time": "2026-07-25T12:00:00+03:00",
  "price": 2500.00,
  "currency": "RUB",
  "status": "CONFIRMED",
  "created_at": "2026-07-20T14:32:15+03:00"
}
```

### Коды ответов

| Код                       | Когда возникает                                            | Описание                                                                              |
| ------------------------- | ---------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| 201 Created               | Бронирование успешно создано                               | Возвращается информация о созданном бронировании                                      |
| 400 Bad Request           | Переданы некорректные входные данные                       | Например, отсутствует обязательное поле или `end_time` меньше либо равно `start_time` |
| 401 Unauthorized          | Пользователь не аутентифицирован                           | Отсутствует или недействителен токен авторизации                                      |
| 404 Not Found             | Рабочее пространство не найдено                            | Переданный `workspace_id` не существует                                               |
| 409 Conflict              | Выбранный временной интервал недоступен                    | Рабочее пространство уже забронировано на пересекающийся временной интервал           |
| 422 Unprocessable Entity  | Запрос синтаксически корректен, но нарушает бизнес-правила | Например, бронирование создаётся на недоступное по расписанию время                   |
| 500 Internal Server Error | Внутренняя ошибка системы                                  | Ошибка при обработке запроса                                                          |

## 3.2. Получение информации о бронировании

### Общая информация

| Поле              | Описание                                                               |
| ----------------- | ---------------------------------------------------------------------- |
| Endpoint name     | GetBooking                                                             |
| HTTP method       | GET                                                                    |
| URL               | `/api/v1/bookings/{booking_id}`                                        |
| Описание сценария | Возвращает информацию о бронировании по его уникальному идентификатору |

### Входящие данные

| Параметр      | Тип    | Тип данных | Обязательный | Описание                              |
| ------------- | ------ | ---------- | ------------ | ------------------------------------- |
| Authorization | Header | string     | да           | Токен авторизации пользователя        |
| booking_id    | Path   | UUID       | да           | Уникальный идентификатор бронирования |

### Пример запроса

Тело запроса отсутствует.

```http
GET /api/v1/bookings/b52cb10b-ef43-4f23-b226-cf44c93f73c1
```

### Успешный ответ

HTTP Status: `200 OK`

| Параметр     | Тип данных         | Обязательный | Описание                              |
| ------------ | ------------------ | ------------ | ------------------------------------- |
| booking_id   | UUID               | да           | Уникальный идентификатор бронирования |
| workspace_id | UUID               | да           | Идентификатор рабочего пространства   |
| start_time   | string (date-time) | да           | Дата и время начала бронирования      |
| end_time     | string (date-time) | да           | Дата и время окончания бронирования   |
| price        | number             | да           | Стоимость бронирования                |
| currency     | string             | да           | Валюта стоимости                      |
| status       | string             | да           | Текущий статус бронирования           |
| created_at   | string (date-time) | да           | Дата и время создания бронирования    |

### Пример успешного ответа

```json
{
  "booking_id": "b52cb10b-ef43-4f23-b226-cf44c93f73c1",
  "workspace_id": "8a12f316-3b1b-4b5f-9b8c-f4d638a7f125",
  "start_time": "2026-07-25T10:00:00+03:00",
  "end_time": "2026-07-25T12:00:00+03:00",
  "price": 2500.00,
  "currency": "RUB",
  "status": "CONFIRMED",
  "created_at": "2026-07-20T14:32:15+03:00"
}
```

### Коды ответов

| Код                       | Когда возникает                                        | Описание                                             |
| ------------------------- | ------------------------------------------------------ | ---------------------------------------------------- |
| 200 OK                    | Бронирование найдено                                   | Возвращается информация о бронировании               |
| 400 Bad Request           | Идентификатор имеет некорректный формат                | Например, `booking_id` не соответствует формату UUID |
| 401 Unauthorized          | Пользователь не аутентифицирован                       | Отсутствует или недействителен токен авторизации     |
| 403 Forbidden             | Пользователь не имеет права просматривать бронирование | Бронирование принадлежит другому пользователю        |
| 404 Not Found             | Бронирование не найдено                                | Бронирование с указанным идентификатором отсутствует |
| 500 Internal Server Error | Внутренняя ошибка системы                              | Ошибка при обработке запроса                         |

## 3.3. Отмена бронирования

### Общая информация

| Поле              | Описание                                                                       |
| ----------------- | ------------------------------------------------------------------------------ |
| Endpoint name     | CancelBooking                                                                  |
| HTTP method       | POST                                                                           |
| URL               | `/api/v1/bookings/{booking_id}/cancel`                                         |
| Описание сценария | Отменяет существующее бронирование, принадлежащее авторизованному пользователю |

### Входящие данные

| Параметр        | Тип    | Тип данных | Обязательный | Описание                                                           |
| --------------- | ------ | ---------- | ------------ | ------------------------------------------------------------------ |
| Authorization   | Header | string     | да           | Токен авторизации пользователя                                     |
| Idempotency-Key | Header | string     | нет          | Уникальный ключ запроса для защиты от повторной обработки операции |
| booking_id      | Path   | UUID       | да           | Уникальный идентификатор бронирования                              |

Тело запроса отсутствует.

### Пример запроса

```http
POST /api/v1/bookings/b52cb10b-ef43-4f23-b226-cf44c93f73c1/cancel
```

### Успешный ответ

HTTP Status: `200 OK`

| Параметр     | Тип данных         | Обязательный | Описание                              |
| ------------ | ------------------ | ------------ | ------------------------------------- |
| booking_id   | UUID               | да           | Уникальный идентификатор бронирования |
| status       | string             | да           | Новый статус бронирования             |
| cancelled_at | string (date-time) | да           | Дата и время отмены бронирования      |

### Пример успешного ответа

```json
{
  "booking_id": "b52cb10b-ef43-4f23-b226-cf44c93f73c1",
  "status": "CANCELLED",
  "cancelled_at": "2026-07-22T09:15:37+03:00"
}
```

### Коды ответов

| Код                       | Когда возникает                                   | Описание                                              |
| ------------------------- | ------------------------------------------------- | ----------------------------------------------------- |
| 200 OK                    | Бронирование успешно отменено                     | Возвращается актуальный статус бронирования           |
| 400 Bad Request           | Идентификатор имеет некорректный формат           | Переданный `booking_id` не соответствует формату UUID |
| 401 Unauthorized          | Пользователь не аутентифицирован                  | Отсутствует или недействителен токен авторизации      |
| 403 Forbidden             | Пользователь не имеет права отменять бронирование | Бронирование принадлежит другому пользователю         |
| 404 Not Found             | Бронирование не найдено                           | Бронирование с указанным идентификатором отсутствует  |
| 409 Conflict              | Бронирование нельзя отменить в текущем состоянии  | Например, бронирование уже завершено                  |
| 500 Internal Server Error | Внутренняя ошибка системы                         | Ошибка при обработке запроса                          |
